# Security Policy

## 지원 버전

MVP 기간에는 최신 `0.1.x` 버전만 보안 수정 대상입니다.

## 취약점 신고

보안 취약점은 공개 issue에 올리지 마세요. 저장소에서 GitHub Security Advisory의 **Report a vulnerability** 기능을 사용할 수 있다면 이를 이용하고, 사용할 수 없다면 관리자에게 비공개 연락 수단을 요청하세요.

신고에는 다음 정보를 포함해 주세요.

- 영향을 받는 버전과 운영체제
- 재현 단계와 예상 영향
- 가능한 경우 최소 재현 파일 또는 로그
- 임의 실행이나 개인정보 노출 가능성

실제 사용자 데이터나 악성 실행 파일은 첨부하지 마세요. 관리자는 접수 확인, 영향 평가, 수정 및 공개 일정을 신고자와 조율합니다.

## 범위

특히 임의 명령 실행, URL 프로토콜 우회, 템플릿 검증 우회, 경로 검증 우회, SQLite 데이터 손상과 Tauri capability 권한 상승을 중요하게 다룹니다.
